# ぷよぷよ通 ゲームルール調査メモ

本ドキュメントは、ぷよぷよ通のゲーム仕様をまとめた調査メモです。初手研究チャートの開発にあたり、チーム内で前提知識を共有することを目的としています。

---

## 1. 盤面の大きさ

- 通常ルールでは、フィールドは **横6列 × 縦12段**（可視領域）で構成される
  - ちびぷよ、でかぷよなどの特殊ルールでは、フィールドサイズが異なる場合がある
- 可視領域の上に **非表示の領域**が存在する
  - ぷよは13段目以上にも積むことができるが、画面上には表示されない
  - 13段目のぷよは連鎖の連結判定に**含まれない**（消えない）
  - ただし、13段目のぷよが落下して12段目以下に入れば通常通り連結判定される
  - 14段目以上のぷよは落下もできず衝突もしないため、実質的には13段目が上限となる
- 座標系: 列は左から1〜6、段は下から1〜13で表現されることが多い

```
       1列  2列  3列  4列  5列  6列
14段 |    |    |    |    |    |    |  ← 非表示行（落下も衝突もしない）
13段 |    |    |    |    |    |    |  ← 非表示行（落下するが連結判定に含まれない）
12段 |    |    |    |    |    |    |
11段 |    |    |    |    |    |    |
 :   |    |    |    |    |    |    |
 2段 |    |    |    |    |    |    |
 1段 |    |    |    |    |    |    |
     +----+----+----+----+----+----+
```

## 2. ぷよの色数

- ぷよぷよ通では、ルールによって使用される色ぷよの数が異なる
  - 甘口: 3色（赤・緑・青）
  - **中辛: 4色**（赤・緑・青・黄）
  - 辛口: 5色（赤・緑・青・黄・紫）
- おじゃまぷよ（白）は色ぷよとは別カテゴリで、連鎖には関与しない
- 色ぷよのみが連結判定の対象となる

## 3. 組ぷよの色数制限（配ぷよルール）

- 1つの組ぷよは **軸ぷよ**（操作の中心）と **子ぷよ**（軸に隣接）の2つで構成される
- ぷよぷよ通の配ぷよには以下のルールが存在する:

### 初手の制限

- **1手目**: 必ず **3色目までの色** から選ばれる（4色目は出現しない）
- **2手目**: 必ず **3色目までの色** から選ばれる（1手目と同じ制限）
- **3手目以降**: 4色すべてが出現しうる

### プール方式の配色アルゴリズム

- ぷよぷよ通（アーケード版）では、配ぷよの色は **プール方式** で決定される
- **256個のぷよ（= 128組）** を1プールとして事前に生成し、シャッフルして配ぷよ順を決定する
- プールの最初の2組（1手目・2手目）は3色制限が適用されたものに上書きされる
- プールを使い切ると次のプールが生成される
- この方式により、長期的な色の出現頻度はほぼ均等になるが、短期的には偏りが生じうる

## 4. ネクスト・ネクネク

- ゲーム中、次に落ちてくる組ぷよ（**ネクスト**）とその次（**ネクネク**）が画面右側に表示される
- これにより、操作中の組ぷよに加えて **2手先まで**（ネクスト + ネクネクの計2組）を確認可能
- 初手研究においては、ネクスト・ネクネクの組み合わせパターンに応じた分岐を管理することが重要
- ネクスト表示は上がネクスト（次の手）、下がネクネク（その次の手）

## 5. 自由落下

### 操作中の落下

- 組ぷよは一定速度で自動的に下方向へ落下する
- プレイヤーは下入力で **高速落下**（ソフトドロップ）が可能
- 一部ルールでは即座に設置する **ハードドロップ** も存在するが、ぷよぷよ通では基本的にソフトドロップのみ

### 設置後の落下

- 組ぷよが設置された後、**ちぎり**（後述）により分離したぷよは自由落下して下に積まれる
- 連鎖処理中に消滅が発生した場合、上にあるぷよは重力により下に落下する
- 落下アニメーション完了後に次の連結判定が行われる（連鎖の連続処理）

### 落下ボーナス

- ぷよぷよ通では、組ぷよを素早く設置すること（高速落下）による **落下ボーナス** が得点に加算される
- 落下ボーナスは連鎖の得点とは別に計算される
- 1段分の落下ごとに1点が加算される（1つの組ぷよあたり最大11点）

## 6. ぷよが消える条件

- **同じ色のぷよが4つ以上連結**（上下左右に隣接）すると消滅する
- 斜めの隣接は連結と見なされない
- 連結判定は **12段目以下** のぷよのみが対象（13段目以上は対象外）
- おじゃまぷよは色ぷよの消滅に巻き込まれて消える（隣接するおじゃまぷよが同時に除去される）
  - ただし、おじゃまぷよ同士では連結判定は行われない

### 連鎖の流れ

1. 組ぷよが設置される
2. ちぎり処理（分離したぷよの落下）
3. 全てのぷよが着地した後、連結判定を行う
4. 4つ以上の連結グループがあれば消滅 → 隣接するおじゃまぷよも除去
5. 消滅後、浮いたぷよが落下する
6. 再び連結判定を行う（**連鎖数 +1**）
7. 消滅が発生しなくなるまで 4〜6 を繰り返す

## 7. 連鎖と点数計算

### 連鎖消去点の公式

```
連鎖消去点 = (消したぷよ数 × 10) × (連鎖ボーナス + 連結ボーナス + 色数ボーナス)
```

ただし、ボーナス合計が **0の場合は1として扱う**（最低保証）。
また、ボーナス合計の **上限は999**。

- 落下ボーナス、全消しボーナスと合わせておじゃまぷよの計算に利用される（後述）

### 連鎖ボーナス

| 連鎖数 | ボーナス |
| ------ | -------- |
| 1      | 0        |
| 2      | 8        |
| 3      | 16       |
| 4      | 32       |
| 5      | 64       |
| 6      | 96       |
| 7      | 128      |
| 8      | 160      |
| 9      | 192      |
| 10     | 224      |
| 11     | 256      |
| 12     | 288      |
| 13     | 320      |
| 14     | 352      |
| 15     | 384      |
| 16     | 416      |
| 17     | 448      |
| 18     | 480      |
| 19     | 512      |

- 理論上、19連鎖よりも大きな連鎖数にすることはできない（6×13÷4 = 19.5）ため、19連鎖が最大の連鎖ボーナスとなる

### 連結ボーナス

| 連結数    | ボーナス |
| --------- | -------- |
| 4（最小） | 0        |
| 5         | 2        |
| 6         | 3        |
| 7         | 4        |
| 8         | 5        |
| 9         | 6        |
| 10        | 7        |
| 11以上    | 10       |

- 1回の消滅で複数の連結グループが同時に消える場合、それぞれの連結ボーナスが合算される

### 色数ボーナス

| 同時消し色数 | ボーナス |
| ------------ | -------- |
| 1色          | 0        |
| 2色          | 3        |
| 3色          | 6        |
| 4色          | 12       |
| 5色          | 24       |

## 8. おじゃまぷよ

### 得点からおじゃまぷよへの変換

- おじゃまぷよの数は以下の式で計算される:

```
得点 = 連鎖消去点 + 落下ボーナス + 全消しボーナス + 剰余点
おじゃまぷよ数 = 得点 ÷ おじゃまレート
```

- **おじゃまレート** はぷよぷよ通では **70** が標準
- 一定時間（マージンタイム）が経過するとおじゃまレートが下がり始める
  - ぷよぷよ通では、マージンタイムは通常96秒に設定されている
- 落下ボーナスはぷよが消えるまでに蓄積された点数が計上され、消すたびに0にリセットされる
- おじゃまレートに達しなかった残りの点数は剰余点として次の連鎖に持ち越される

### 相殺（そうさい）

- ぷよぷよ通から導入されたルール
- 相手からおじゃまぷよが予告されている状態で自分が連鎖を起こすと、自分の連鎖で生成されたおじゃまぷよ数で相手の予告おじゃまを **相殺** できる
- 相殺後に余剰があれば、その分が相手への攻撃となる
- 相殺は連鎖が完了するまでリアルタイムに処理される

### 予告ぷよ

- フィールド上部に、降ってくる予定のおじゃまぷよの数が **予告シンボル** として表示される
- 予告シンボルは最大6個まで表示され、量に応じてより大きなシンボルにまとめられる
- SEGA版ぷよぷよ通で使用されるシンボルと対応するおじゃまぷよ数:

| シンボル | おじゃまぷよ数 | 備考                   |
| -------- | -------------- | ---------------------- |
| 小ぷよ   | **1個**        | 最小単位               |
| 大ぷよ   | **6個**        | 1段分に相当            |
| 岩ぷよ   | **30個**       | 5段分。1度に降る最大量 |
| 星ぷよ   | **180個**      |                        |
| 月ぷよ   | **360個**      |                        |
| 王冠ぷよ | **720個**      |                        |

- AC版では、岩ぷよの次はきのこぷよ（200個）→ 星ぷよ（300個）→ 王冠ぷよ（400個）だった
- 連鎖中に予告ぷよが発生した場合、シンボルがリアルタイムに更新される
- 相殺によって予告ぷよが減少すると、シンボルも即座に減少する

### おじゃまぷよの落下パターン

- おじゃまぷよは **1段（6個）** を単位として降る
- 1段未満の端数は、ランダムな列に配置される
- 1ターンで降るおじゃまぷよの数には上限がある（最大 **30個 = 5段**）
  - 残りは次のターン以降に持ち越される

### 全消しボーナス

- フィールド上のすべてのぷよ（色ぷよ・おじゃまぷよ）を消し切ると **全消し** となる
- ぷよぷよ通では、全消し後の **次の連鎖** で **おじゃまぷよ30個分（= 5段分）** のボーナスが加算される
- 初手研究においては、少ない手数で全消しを狙える手順や、全消し時により多くのおじゃまぷよを送れる手順が重要な研究対象となる

## 9. ゲームオーバー条件

- 組ぷよの初期出現位置は **3列目** の上部
  - 軸ぷよ: 3列目12段目
  - 子ぷよ: 3列目13段目
- **3列目の12段目** が埋まっている状態で次の組ぷよが出現しようとすると、出現に失敗して **窒息**（ゲームオーバー）となる
- 連鎖処理中に3列目が一時的に埋まっていても、連鎖完了後に空きができれば窒息にはならない

```
       1列  2列  3列  4列  5列  6列
13段 |    |    | 子 |    |    |    |  ← 子ぷよの初期位置
12段 |    |    | 軸 |    |    |    |  ← 軸ぷよの初期位置（ここが埋まると窒息）
11段 |    |    |    |    |    |    |
 :
```

## 10. ぷよの回転

### 基本操作

- 組ぷよは時計回り（右回転）・反時計回り（左回転）に **90度ずつ回転** できる
- 組ぷよの2つのぷよのうち、**軸ぷよ** を中心として **子ぷよ** が回転する
- 回転先にぷよや壁がある場合、以下の補正ルールが適用される

### 壁蹴り

- 回転先が **壁**（フィールド端）によって塞がれている場合、組ぷよ全体が壁と反対方向に **1マス押し出されて** 回転が成立する
- 例: 左壁に接した状態で子ぷよを左に回転しようとすると、組ぷよ全体が右に1マスずれて回転する
- 壁蹴りの先もぷよで塞がれている場合、壁蹴りは不成立となり、代わりにクイックターン判定に移行する

### 床蹴り

- 回転先が **床や積まれたぷよ**（下方向の障害物）によって塞がれている場合、組ぷよ全体が **上方向に1マス押し上げられて** 回転が成立する
- 例: 横向きの組ぷよが地面に接している状態で子ぷよを下に回転しようとすると、組ぷよ全体が1段上にずれて回転する
- 床蹴りは **階段のぼり**（後述）など多くのまわしテクニックの基礎となる

### クイックターン

- 組ぷよが左右をぷよや壁に挟まれて90度回転できない場合、回転ボタンを **2回入力** することで **180度回転**（上下反転）が行える
- ぷよぷよ通以降のすべてのシリーズで使用可能
- 挟まれた狭い空間で軸ぷよと子ぷよの上下を入れ替えたい場合に使用する

### 昇竜ぷよ

- 床蹴りを連続で利用して、段差状に積まれたぷよの上を組ぷよで **乗り越えていく** テクニック
- 回転と横移動を交互に繰り返すことで、1段ずつ階段を登るように組ぷよを移動させる
- 方法1: 同じ回転ボタンを連続で素早く入力しながら横入力
- 方法2: 時計回りと反時計回りを交互に入力しながら横入力（こちらの方が成功率が高い）
- 12段目に置かれたぷよを超えて向こうに組ぷよを配置する壁超えや13段目以上へのぷよ配置も可能となり、特に終盤における不要なぷよを14段目に置いて組ぷよを引き直すテクニックが有効

### ちぎり

- 組ぷよが設置されるとき、軸ぷよと子ぷよの下の地形が異なる高さの場合、組ぷよは分離される（ちぎり）
- ちぎりが発生すると、高い方のぷよが先に着地し、もう一方が落下する
- ちぎりには落下距離に応じた時間がかかるため、初手研究ではちぎりを最小化する手順が重視される
